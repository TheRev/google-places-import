Google Places Directory - Business Map Shortcode Test

Note: This is a test page for the [gpd-business-map] shortcode fix. To properly test this, copy the shortcode examples below into a WordPress page or post.

Test 1: Basic Map

This should display a map with all businesses that have coordinates:

[gpd-business-map]

Expected result: Interactive map showing business markers with default height (400px) and zoom level (14).

Test 2: Custom Height and Zoom

This should display a taller map with different zoom level:

[gpd-business-map height="500px" zoom="10"]

Expected result: Map with 500px height and zoom level 10.

Test 3: Category Filter

This should display only businesses from a specific category:

[gpd-business-map category="restaurant" height="400px"]

Expected result: Map showing only restaurants (if any exist with that category).

Test 4: Single Business Map

This should display a map focused on a specific business:

[gpd-business-map id="123" zoom="16"]

Expected result: Map centered on business with ID 123 (replace with actual business ID).

Test 5: No Clustering

This should disable marker clustering:

[gpd-business-map clustering="false"]

Expected result: Map with individual markers instead of clusters.

Troubleshooting

If the shortcode doesn't work:

  1. Check registration: Verify that the shortcode is registered in class-gpd-shortcodes.php line ~29
  2. Check businesses: Ensure you have businesses imported with coordinate data (_gpd_latitude and _gpd_longitude)
  3. Check assets: Verify Leaflet CSS/JS files are loading properly
  4. Browser console: Check for JavaScript errors in the browser developer tools
  5. WordPress debug: Enable WP_DEBUG to see any PHP errors

Error Messages:

  • "No businesses with location data found" - No businesses have latitude/longitude data
  • "Leaflet map initialization function not found" - JavaScript files not loading
  • Blank output - Shortcode not registered properly
